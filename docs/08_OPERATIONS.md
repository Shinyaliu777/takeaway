# 运维与发布

更新时间：2026-03-23

## 1. 每次发布标准步骤

1. 本地改代码
2. 本地校验
3. 提交并推送 `main`
4. 微信云托管从 `main` 发版
5. 用户端和商家端清缓存并重新编译
6. 如线上行为异常，手动重启 `takeaway-api`

## 2. 每次发布后建议验证

### 用户端

- 登录 / 游客浏览
- 首页菜单加载
- 首页付款方式弹层
- 购物车核对
- 创建订单
- 上传付款截图
- 订单详情状态文案

### 商家端

- 登录
- 首页工作台
- 订单审核
- 商品管理
- 规则管理
- 店铺设置上传收款码

## 3. 每次发布后建议核对的数据

- `shop.wechat_qr_url / alipay_qr_url / tng_qr_url`
- `order`
- `paymentorder`
- `paymentorder.proof_image_url`

## 4. 当前常见问题

### 看见旧 UI

通常是微信开发者工具缓存。

处理方式：

- 清缓存并重新编译

### 云托管规则未生效

通常是云端还没拉到最新 `main`，或者当前实例没切过去。

处理方式：

- 确认当前运行 revision
- 手动重启 `takeaway-api`
- 再看运行日志

### 登录转圈

优先排查：

- `/api/user/login`
- 云托管日志
- 小程序请求域名

### 收款码或付款截图有 URL 但图片打不开

优先排查：

- `shop.wechat_qr_url / alipay_qr_url / tng_qr_url`
- `paymentorder.proof_image_url`
- 是否仍指向旧 `/uploads/...`
- 当前服务实例是否刚发版 / 是否已经重启

### `/api/orders/create` 线上 500

当前经验结论：

- 不要先假设是数据库结构问题
- 先确认线上运行代码是否已切到最新
- 再看 `create_order_failed` 日志

## 5. 运维排查入口

- 云托管运行日志
- 云托管 MySQL / DMS
- 微信开发者工具 Network / Console

## 6. 回归推荐顺序

1. 首页是否能看到付款方式入口
2. 收款码弹层是否能显示图片
3. 创建订单是否成功
4. 订单详情是否能上传付款截图
5. 商家端是否能确认付款
6. 商家端是否能推进到配送
