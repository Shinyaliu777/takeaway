# 产品说明

更新时间：2026-03-22

## 1. 产品定位

这是一个面向单店场景的微信外卖点餐系统，采用：

- 用户端小程序点餐
- 商家端小程序管理
- 线下转账 + 上传付款截图 + 商家审核

## 2. 用户端当前流程

1. 用户进入小程序
2. 必须先登录
3. 登录后进入菜单页
4. 用户直接选择具体菜品
5. 系统实时按荤素数量拆分最优套餐
6. 购物车页只负责核对
7. 创建订单
8. 上传付款截图
9. 等待商家确认到账

## 3. 菜单模型

菜单当前展示的是具体菜品，不是固定套餐商品。

分类上分为：

- 荤菜
- 素菜
- 单点主食 / 额外米饭

补充规则：

- 同一道菜可以重复选
- 荤菜 / 素菜不能单独结算
- 必须完整组成套餐才能下单
- 套餐默认含 1 份米饭
- 额外米饭按后台价格单独计费

## 4. 当前套餐计价规则

默认规则如下：

- 两荤一素：`20.8`
- 一荤两素：`15.8`
- 一荤一素：`14.8`
- 两荤：`18.8`
- 两素：`9.9`
- 两荤两素：`22.8`
- 三素：`11.8`
- 三荤：`26.8`

这些规则现在已经后台化，商家端可以直接修改。

## 5. 时段逻辑

商品支持：

- 午餐可点
- 晚餐可点

用户端菜单页会根据午餐 / 晚餐切换过滤商品。

## 6. 购物车和订单展示

购物车页当前展示：

- 已选菜品
- 套餐拆分结果
- 每个套餐内包含哪些具体菜
- 额外单点项
- 总价
- 地址

订单页也沿用这个思路，避免只显示一个抽象套餐名。

## 7. 支付与履约

当前不是自动支付模式，业务流程是：

1. 用户下单
2. 用户线下转账
3. 用户上传付款截图
4. 商家审核
5. 商家确认到账
6. 商家推进配送中 / 已完成

## 8. 商家端核心能力

商家端当前可管理：

- 商品
- 分类
- 商品午餐 / 晚餐可点
- 店铺信息
- 收款码
- 推荐位
- 套餐规则
- 套餐价格
- 额外米饭价格
- 用户中心
- 历史订单查看
