## 5. 订单状态验收建议

### 场景 1：创建订单
- 用户端加入 `香菇鸡 x2`
- 进入购物车
- 选择默认地址
- 创建订单
- 预期
  - 新订单生成
  - 状态：`PENDING_PAYMENT / UNPAID`

### 场景 2：上传付款截图
- 用户端进入订单详情
- 查看付款方式
- 上传测试截图
- 预期
  - 状态变为：`PAYMENT_REVIEW / PROOF_UPLOADED`

### 场景 3：商家确认付款
- 商家端进入订单详情
- 查看付款截图
- 点击确认到账
- 预期
  - 状态变为：`PAID / SUCCESS`

### 场景 4：开始配送
- 商家端在已付款订单上点击配送
- 预期
  - 状态变为：`DELIVERING / SUCCESS`

### 场景 5：完成订单
- 商家端将配送中订单改为完成
- 预期
  - 状态变为：`COMPLETED / SUCCESS`

## 6. 测试截图要求

用于联调时，付款截图可以使用：
- 本地测试图片
- 任意非真实敏感支付凭证

要求：
- 避免真实银行卡号
- 避免真实完整身份信息
- 能明显看出“付款成功”即可
